# Weekly Review

## Purpose

Produce a consistent weekly digest of the Personal AI system so follow-ups,
drift, and triage items surface on a regular cadence. The review is a prompt to
act, not another place to store durable notes.

## How It Works

The review is generated by a deterministic local script so output is stable and
reproducible across both Codex and Claude. Prefer the script over hand-writing
the review.

## Procedure

1. Run the generator from the workspace root:

   ```bash
   python tools/weekly_review.py
   ```

2. The script writes `reviews/YYYY-MM-DD-weekly-pai-review.md` and
   prints the path. It reads `people/active/`, `work/active/`,
   `work/completed/`, `memory/`, `inbox/README.md`,
   `assistant/current-context.md`, and `background/projects.md`. It keeps prior
   reviews; nothing is deleted or committed automatically.
3. Open the generated file and act on the "Suggested Attention This Week" items:
   follow up on stale people, triage inbox ideas, repair missing memory paths,
   merge duplicate decisions, and review recently completed work for durable
   lessons.
4. Close the loop. When the review flags something, resolve it (or consciously
   defer it) so the same flag does not recur week after week.

## Inputs And Configuration

- Person files must keep an accurate `## Last Call` `Date:` so follow-up timing
  is correct.
- Work files should follow `work/README.md`'s template (`Status:`, `Updated:`,
  `## Desired Outcome`, `## Next Action`) so the review can summarize them.

## Output

A single dated Markdown review under `reviews/`.

## Verification

- The dated review file exists at the printed path.
- People follow-up timing matches the `Last Call` dates in `people/active/`.
- Flagged items are acted on or deliberately deferred, not ignored.

## Agent Notes

- Codex and Claude both run the same Python generator; prefer it over
  re-implementing the review by hand.
- If Python is not available on the machine, do not install anything. Instead,
  read the same source files yourself and produce a review with the same
  sections, clearly noting it was generated manually.
- Do not store durable notes in the review. Promote durable lessons to
  `memory/`, relationship updates to `people/`, and context to `background/`.
- If the generator errors or a referenced path is missing, report it rather than
  producing a partial review presented as complete.
